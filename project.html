STIP Sipencatar 🎓

About Project

A web-based admission system that manages the entire cadet recruitment process, including applicant registration, document submission, admission selection, examination schedules, and announcement of admission results.

The system supports two primary user roles:

👤 Prospective Cadet (Applicant)

Prospective cadets can create an account to participate in the admission process. Through the system, they can:

  • ✔ Create and activate an admission account.
  • ✔ Complete personal and academic information.
  • ✔ Upload all required admission documents.
  • ✔ Apply for the active admission period.
  • ✔ Monitor the status of document verification and selection stages.
  • ✔ View admission results and selection announcements.


👨‍💼 Administrator

Administrators manage the entire admission process through a centralized administration panel with the following modules:

  • ✔ Dashboard – Monitor admission activities and overall system status.
  • ✔ General Reference – Manage master data and general reference information.
  • ✔ Academic Reference – Configure academic data and admission settings.
  • ✔ Master Data – Manage required documents, upload limits, selection stages, examination types, and admission periods.
  • ✔ Selection Management – Verify applicant documents, manage selection stages, and determine successful applicants.
  • ✔ Reports – Generate admission transaction reports and administrative summaries.
  • ✔ System Management – Configure system settings and operational parameters.
  • ✔ User Management – Manage administrator accounts, roles, and access permissions.
  • ✔ Landing Page Management – Update public information, announcements, and website content.
  • ✔ Application Settings – Manage system configuration, administrative requirements, applicant password reset, applicant email removal, and manual applicant account activation.

STIP Register 📝

About Project

A web-based registration system for admitted cadets, supporting document verification, re-registration, virtual account payment, NRP generation, student data management, and administrative reporting.

The system supports two primary user roles:

👤 Cadet (Participant)

Accepted cadets are required to register an account before completing the re-registration process. Through the system, they can:

  • ✔ Create and activate their registration account.
  • ✔ Complete personal, academic, and administrative information.
  • ✔ Upload all required registration documents.
  • ✔ Monitor document verification status.
  • ✔ Receive a BNI Virtual Account (VA) code for registration payment.
  • ✔ Complete the re-registration process for the active admission period.


👨‍💼 Administrator

Administrators manage the entire re-registration process through a centralized administration panel, including:

  • ✔ Dashboard – Monitor registration activities and overall system status.
  • ✔ General Reference – Manage master data and general reference information.
  • ✔ Academic Reference – Configure academic data and registration settings.
  • ✔ Registration – Manage cadet registrations, document verification, NRP generation, and re-registration workflows.
  • ✔ Reports – Generate registration, payment, cadet, and administrative reports.
  • ✔ System Management – Configure system settings and operational parameters.
  • ✔ User Management – Manage administrator accounts, roles, and permissions.
  • ✔ Landing Page Management – Update public website content and announcements.
  • ✔ Application Settings – Manage system configuration, NRP settings, payment types, payment configuration, IP whitelist, print templates, cadet user accounts, password reset, and manual account activation.
  • ✔ BNI Callback Logs – Monitor and validate Virtual Account payment callbacks from BNI integration.

STIP Asset 🏢

About Project

STIP Asset is a web-based public facility reservation system developed for Sekolah Tinggi Ilmu Pelayaran (STIP) Jakarta. The platform enables the public to reserve accommodation and other public service facilities online through a centralized reservation system.

The application simplifies the booking process by allowing users to register, submit reservation requests, select available facilities, and manage their bookings, while providing administrators with tools to efficiently manage assets, reservations, and public service information.


The system supports two primary user roles:

👤 Customer

Customers are required to create an account before making a reservation. Through the system, they can:

  • ✔ Register and activate a customer account.
  • ✔ Browse available public facilities and accommodations.
  • ✔ Select check-in date and reservation duration.
  • ✔ Choose optional services, such as breakfast.
  • ✔ View available rooms, facilities, and booking costs.
  • ✔ Submit online reservation requests.
  • ✔ Review booking details before confirmation.
  • ✔ Cancel reservations according to the cancellation policy.
  • ✔ Receive booking status updates and reservation information.


👨‍💼 Administrator

Administrators manage the entire reservation system through a centralized administration panel with the following modules:

  • ✔ Dashboard – Monitor reservation activities, facility availability, and overall system performance.
  • ✔ User Management – Manage customer accounts and administrator access.
  • ✔ Asset Management – Manage public facilities, accommodation assets, and asset availability.
  • ✔ Asset Locking – Block or reserve assets that are temporarily unavailable for booking.
  • ✔ Reservation Management – Process reservation requests and monitor booking status.
  • ✔ Reservation Cancellation – Review and manage customer cancellation requests.
  • ✔ Cancellation Settings – Configure reservation cancellation policies and rules.
  • ✔ Transaction List – Monitor booking transactions and payment records.
  • ✔ Feedback Management – Manage customer feedback, suggestions, and service evaluations.
  • ✔ Gallery Management – Manage facility images displayed on the website.
  • ✔ News Management – Publish announcements and news updates.
  • ✔ Public Services – Manage public service information and facility details.
  • ✔ System Settings – Configure application settings and operational parameters.
